Card surface system

Five tiers. Every card inherits one.

Surfaces are a brand statement, not a styling choice. The brand book uses five card tiers, each with its own job. Reach for the right tier by the role the card plays — not by what it looks like in isolation. Most cards in the brand book are tier 1 (solid). Tier 2 (heavy obsidian) is reserved for moments. Tiers 4 and 5 (image-backed) are reserved for sections where the imagery itself reinforces meaning.

Tier 1 · Solid
Default body card

When to use: data tables, palette rows, chart cards, default body containers.
Spec: background: var(--surface) + 1px var(--border-light).

Tier 2 · Obsidian heavy
Quotes, North Stars, key callouts

When to use: the moment you want the reader to stop and reread. North Stars, italic pull-quotes, the test-the-values-flow-from line.
Spec: rgba(0,0,0,0.65) + 30px backdrop blur + saturate 1.3 + top-edge inset highlight + drop shadow. Purple left-border 3px solid var(--accent).
Used on: .northstar, .quote-block.

Tier 3 · Obsidian subtle
Body card grids, secondary panels

When to use: grids of related cards where each one needs distinction without dominating. Promise principles, anti-positioning tiles, sub-brand registry items.
Spec: rgba(10,10,10,0.5) + 20px backdrop blur + saturate 1.2.
Used on: .principle.

Tier 4 · Image-backed (28px blur)
Abstract artistic glass — mood as meaning

When to use: sections where colour temperature reinforces the principle being stated. Values cards (warm/cool/minimal triad). Image is reduced to a colour wash — no specific subject visible.
Spec: per-card --scene CSS var → image as ::before at 28px blur + saturate 1.35, scaled out 40px to prevent edge bleed. Dark gradient veil ::after.
Used on: .chart-card.has-scene.

Tier 5 · Image-backed (5px blur, gradient veil)
Editorial photo + caption — image as content

When to use: sections where the image is part of the message — channel previews, sub-brand cards, anywhere the imagery should be recognisable. Image visible at top of card, gradient veil pulls darkness behind text for legibility.
Spec: per-card --scene at 5px blur + saturate 1.1. Gradient veil dark at bottom (0.85) → mid (0.55) → light at top (0.35) so type sits on the darkest band.
Reserved for: Channels section, sub-brand cards (when applied).

Choosing a tier

Default to Tier 1. Most cards should be solid. Backdrop-filter glass is real GPU cost — use it where it earns its place, not as the default.

Tier 2 is a moment. If you find yourself using it more than 2–3 times in a section, you're diluting the impact. The North Star is a moment. Every value isn't.

Tier 3 is for visual distinction in body grids. Three or four related cards that should each feel like a distinct surface, none of them dominating. Use when the felt bg makes the grid feel undifferentiated.

Tier 4 is reserved for principle sections. Values, possibly Stance. The image is reduced past recognition — what reads is the colour. Pair the colour temperature to the meaning (warm = action, cool = transmission, neutral = restraint).

Tier 5 is for sections where the image IS content. Channel previews, sub-brand cards, anywhere a recognisable scene reinforces the message. Don't use for principle statements — the image will fight the type.

Watchpoints: backdrop-filter is unreliable on iOS Safari (sometimes substituted with a static fallback) — test on real device. Image-backed cards add ~1MB per image to page weight — cache aggressively, consider WebP versions for production.

Asset library · how to use

Reference any asset by its ID — AS-14, AS-62, AS-100 — when describing pages, articles, course modules, or products. The ID is stable; the visual may evolve.

All 81 components are hand-coded HTML / CSS / SVG. No JS dependencies, no third-party libraries, no AI imagery. Copy the markup directly from this page's source. Extend the library by adding a new .asset block in the appropriate range — keep AS-XX sequential, leave gaps for siblings.

Numbering ranges (reserved): 01–09 loaders · 10–19 inputs · 20–29 cards · 30–39 overlays · 40–49 data UI · 50–59 motion · 60–69 process · 70–79 matrices · 80–89 hierarchy · 90–99 timeline · 100–109 concepts · 110–119 advanced charts · 120–129 callouts.
